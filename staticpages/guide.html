How to use the IceCube Dispenser

Back to the app

Assumptions

We assume that you have a Raspberry Pi that you would like to use as an EPICS IOC to control/monitor a system via a serial port. For the moment, this port is hard-coded as being /dev/ttyACM0, since this is the standard for Arduino systems, however future updates will allow flexibility in this. To alter the serial port manually, edit the file /home/pi/Apps/epics/iceCubeIOC/iocBoot/iociceCubeIOC/st.cmd and alter the reference to /dev/ttyACM0.
The control functionality offered by this architecture is, from the point of view of the EPICS IOC, a state-machine. That is, the IOC will send serial commands to the controlled-system (e.g., the Arduino) to change its state, or to request the values of various settings. The detailed behaviour of the system is expected to be coded into the system itself; e.g., via code uploaded to the Arduino microcontroller (perhaps modelled on the skeleton code produced by this app).

Installing EPICS

For any of this to work, you will need EPICS installed on your Raspberry Pi. This should be done by following the instructions given by Steve Molloy at this link. Future work will provide functionality to do all of this work for you.

Building an IceCube

Start by downloading the empty IceCube IOC. The link to this is on the home page of this app, or you can just click here. Download that file, and unzip/untar it into some appropriate place. We recommend putting it in /home/pi/Apps/epics/.
tar zxvf iceCubeIOC.tar.gz -C /home/pi/Apps/epics/
There are two files missing from the IceCube IOC that you just downloaded, and the lack of these will mean that an attempt to compile it will fail. You need to make two files:
  • arduino.proto defines the communications protocol.
  • arduino.db specifies process variables that will be served by the IOC.
These files are contructed by filling out the necessary values on the forms on the main page of the app. As you do this, you will notice that the code blocks at the bottom of the page will be populated according to the values you have provided. These blocks contain the necessary contents of the two missing files.
Copy the code listed under "Protocol File" into a new file called /home/pi/Apps/epics/iceCubeIOC/iceCubeIOCApp/Db/arduino.proto, and the code listed under "EPICS DB File" into a new file called /home/pi/Apps/epics/iceCubeIOC/iceCubeIOCApp/Db/arduino.db. Note that the naming and location of these files (with respect to /home/pi/Apps/epics/iceCubeIOC/) is critical.
Once that has been done, navigate to /home/pi/Apps/epics/iceCubeIOC and run make. If this is successful, you are now able to run the IOC.
./bin/linux-arm/iceCubeIOC iocBoot/iociceCubeIOC/st.cmd

Using the Arduino skeleton file

If you want to control an Arduino-based system with this, then you need to make sure the Arduino is set up so as to properly react to the serial commands sent by the IOC. There are many ways to do that, but one way would be to use the suggested skeleton file.
Copy and paste it into the Arduino IDE, add the appropriate code, and upload to the Arduino. This should provide you with a fully functional EPICS-controlled Arduino.
